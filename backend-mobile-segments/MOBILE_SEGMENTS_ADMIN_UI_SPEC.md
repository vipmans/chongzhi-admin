# 号段管理后台页面设计与操作流程

本文档用于补齐“手机号号段库管理”的后台页面设计，适配当前统一管理平台。

目标：

1. 让运营、技术支持、运维可以直观看到号段库是否完整
2. 支持单条新增、编辑、查询、导入、校验、发布
3. 支持追踪每次导入批次和变更结果
4. 保障 `preview-split`、商品匹配、渠道充值预览的前置数据稳定

---

### 2.2 新增 / 编辑号段弹窗

表单字段：

1. `手机号前7位`
   - 必填
   - 只允许 7 位数字
   - 新增时唯一校验
2. `省份`
   - 必填
3. `城市`
   - 选填
4. `运营商编码`
   - 必填
5. `运营商名称`
   - 必填
6. `状态`
   - 必填
   - 默认 `ACTIVE`
7. `数据来源`
   - 选填
   - 如：`MANUAL` / `IMPORTED` / `THIRD_PARTY`
8. `来源版本`
   - 选填
   - 如：`2026-04`
9. `备注`
   - 选填

校验规则：

- `手机号前7位` 必须匹配 `^\d{7}$`
- `运营商编码` 必须在系统枚举内
- `运营商名称` 与编码建议自动联动填充
- 已存在号段禁止重复创建

弹窗按钮：

- `取消`
- `保存`

3. `新增数量`
4. `更新数量`
5. `未变化数量`

7. `错误数量`

## 5. 操作流程

### 5.1 单条补号段流程

适用场景：

- 某个手机号预览失败
- 明确只缺少个别前缀

流程：

1. 运营或技术支持拿到失败手机号
2. 截取前 7 位
3. 在 `号段列表` 搜索该前缀
4. 若不存在，点击 `新增号段`
5. 填写省份 / 城市 / 运营商
6. 保存后重新调用 `preview-split`
7. 若预览恢复正常，则记录处理完成

### 5.2 批量导入流程

适用场景：

- 上游提供新版本全量号段库
- 需要周期性补齐

流程：

1. 运维进入 `批量导入`
2. 上传 CSV
3. 先执行 `上传预检`
4. 查看新增 / 更新 / 删除 / 错误统计
5. 无误后点击 `确认导入`
6. 导入完成后进入 `导入记录`
7. 抽样校验若干号段
8. 再联调 `preview-split`

### 5.3 回滚流程

适用场景：

- 导入数据源错误
- 省份/运营商大范围异常

流程：

1. 在 `导入记录` 找到异常批次
2. 查看差异影响范围
3. 点击 `回滚到上一版本`
4. 系统按 `before_json` 恢复
5. 记录回滚日志
6. 重新执行异常校验

## 6. 页面交互建议

为了让运营更容易定位充值失败问题，页面上建议增加这些体验：

1. 在号段列表搜索框中支持直接输入完整 11 位手机号
   - 系统自动截取前 7 位查询
2. 在详情页显示“该号段是否可用于充值路由”
3. 在异常页直接展示：
   - `后端号段库未收录`
   - `已识别运营商但未识别省份`
   - `号段存在但状态为停用`
4. 在导入预检时展示“本次将影响多少充值商品路由”

## 7. 与充值业务的联动建议

建议在订单预览失败时，把错误归因打通到后台：

### 7.1 预览失败分类

1. `SEGMENT_NOT_FOUND`
   - 号段不存在
2. `CARRIER_ONLY_MATCHED`
   - 只识别到运营商，未识别到省份
3. `PRODUCT_NOT_FOUND`
   - 号段识别成功，但商品未配置
4. `SUPPLIER_NOT_FOUND`
   - 商品存在，但供应商映射未配置

### 7.2 后台联动动作

在异常页中可直接提供：

- `去新增号段`
- `去商品配置`
- `去供应商映射`
- `去查看渠道授权`

这样定位问题会快很多。

## 9. 推荐验收点

1. 输入一个不存在的前 7 位，能否快速新增
2. 导入错误 CSV，是否能返回具体错误行
3. 导入正确 CSV，是否能正确统计新增 / 更新 / 删除
4. 导入后 `preview-split` 是否恢复
5. 回滚后数据是否恢复
6. 变更日志是否完整
